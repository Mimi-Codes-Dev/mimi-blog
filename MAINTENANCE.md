# 🛠️ 米米部落格維護指南 (Mimi's Blog Maintenance Guide)

此文件專門用於記錄 `mimi-blog-repo` 的長期維護要點，確保部落格在內容增加後仍能保持穩定與一致性。

## 📋 維護核心目標
- **一致性**：確保所有文章的 HTML 結構統一，視覺風格保持「米米色調」。
- **正確性**：確保 `index.html` 的索引連結與 `posts/` 資料夾中的檔案完全對應。
- **可用性**：確保網站內容在不同裝置上均能正確渲染。

---

## 📂 維護項目

### 1. 內容完整性檢查 (Content Integrity)
每當新增或修改文章後，應執行以下檢查：
- [ ] **路徑核對**：檢查 `index.html` 中的 `<a href="posts/...">` 路徑是否與實際檔案名稱一致。
- [ ] **標籤閉合**：檢查新撰寫的 HTML 文章是否所有標籤（如 `<div>`, `<p>`, `<a>`）均正確閉合，避免導致頁面崩潰。
- [ ] **簡介更新**：確保 `index.html` 中的文章簡介能準確概括文章內容。

### 2. 視覺與風格維護 (Style Maintenance)
- **CSS 統一**：所有樣式修改應在 `style.css` 中統一完成，避免在 HTML 中使用 inline style。
- **色調把關**：維持米米專屬的粉嫩視覺風格，任何重大色調更動需記錄在此指南中。

### 3. 部署流程維護 (Deployment)
維護標準的 Git 同步流程，避免直接在 GitHub Web UI 修改導致版本衝突：
```bash
git add .
git commit -m "🔧 維護: [描述維護內容]"
git push origin main
```

### 4. 定期清理與優化 (Cleanup & Optimization)
- **過時內容**：定期審視 `posts/`，將不再適用的內容移動至 `archive/`（若未來建立該資料夾）。
- **效能檢查**：檢查圖片大小（若有加入圖片）以確保頁面載入速度。

---

## ⚠️ 緊急處理
若發現網站出現渲染錯誤（如排版亂掉、連結 404）：
1. 優先檢查最近一次的 `git commit` 變更。
2. 回到 `README.md` 確認專案結構是否被意外修改。
3. 若無法修復，可使用 `git checkout` 回退至上一個穩定版本。
